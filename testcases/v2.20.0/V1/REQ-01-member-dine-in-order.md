## Overview

会员（H5/移动端）通过扫描桌码进入点餐页面，完成浏览 → 下单 → 接单 → 支付 → 取餐全流程。

支持两种业务模式：
- **即时支付模式**：先支付后接单
- **先下单后付模式 (Order-First-Pay-Later)**：先下单，接单备餐完成后支付

---

## 1. User Stories

### US-01.01 商品浏览

**As a** 顾客（会员端用户）
**I want to** 通过扫描桌码进入点餐页面并按堂食/到店自取过滤商品
**So that** 我只看到当前用餐方式可用的商品

### US-01.02 下单

**As a** 顾客
**I want to** 选择商品后提交堂食订单，支持选酱料和套餐加购
**So that** 我的订单完整反映我的需求

### US-01.03 先下单后付

**As a** 顾客
**I want to** 先提交订单等商家确认后再付款
**So that** 我不用在商家可能拒单的情况下提前付款

### US-01.04 自动接单

**As a** 商家
**I want to** 扫码点餐订单自动接单，无需人工审核
**So that** 减少顾客等待时间，提升翻台率

### US-01.05 支付

**As a** 顾客
**I want to** 通过 QR PromptPay 等方式完成支付
**So that** 我可以方便快捷地完成付款

### US-01.06 订单详情

**As a** 顾客
**I want to** 查看订单的折扣明细、赠菜/退菜状态、多支付方式
**So that** 我清楚了解每一笔费用的来源

### US-01.07 取消订单

**As a** 顾客
**I want to** 在商家未接单前取消订单并获得退款
**So that** 我不会为不需要的订单付款

### US-01.08 整单备注

**As a** 顾客/收银员
**I want to** 会员端和收银端的整单备注实时同步
**So that** 双方都能看到完整的备注信息

## 3. Business Rules

| Rule ID | Rule | Description |
|---------|------|-------------|
| BR-01.01 | 两阶段下单 | 订单必须经过 create → submit 两步，不能直接提交 |
| BR-01.02 | 班次分配优先级 | 自动接单时：主收银机班次 → 最先登录的收银机班次 → 留空 |
| BR-01.03 | 禁止拆单 | 会员端堂食订单不允许拆分 |
| BR-01.04 | 抹零规则关闭 | 会员端堂食订单不适用抹零，金额精确计算 |
| BR-01.05 | 最小支付金额 | 订单金额 < 1 时禁止支付 |
| BR-01.06 | 送厨减库存 | 所有商品接单时强制扣减库存 |
| BR-01.07 | QR PromptPay 不可拒单 | 使用 QR PromptPay 支付的订单不允许拒单 |
| BR-01.08 | 营业时间校验 | 不在营业时间内不允许创建订单 |
| BR-01.09 | KDS 可选 | 未开启 KDS 时跳过备餐状态，不阻塞流转 |

## 5. Acceptance Criteria

### AC-01.01 即时支付模式完整流程

- **Given** 商家已开启门店扫码点餐
- **When** 顾客扫描桌码 → 选择商品 → 提交订单 → 支付成功
- **Then** 订单状态变为"已支付"，收银端和 KDS 同步显示，库存扣减，ERP SI+PE 生成

### AC-01.02 先下单后付模式完整流程

- **Given** 商家已开启先下单后付模式 (`is_order_first_pay_later = true`)
- **When** 顾客提交订单 → 系统自动接单 → 备餐完成 → 顾客支付
- **Then** 订单从"待接单" → "备餐中" → "待支付" → "已完成"，每个状态转换正确触发统计和 ERP 同步

### AC-01.03 班次分配

- **Given** 会员端堂食订单自动接单
- **When** 系统查找可用班次
- **Then** 按优先级分配：主收银机当班号 → 最先登录的收银机班次 → 留空

### AC-01.04 拒单退款

- **Given** 商家拒绝会员端堂食订单
- **When** 拒单操作完成
- **Then** 自动发起退款，顾客商品列表恢复，订单状态正确

### AC-01.05 整单备注同步

- **Given** 顾客在会员端添加整单备注
- **When** 提交订单
- **Then** 收银端订单详情显示相同备注，反之亦然

### AC-01.06 折扣明细

- **Given** 订单使用了会员折扣
- **When** 查看订单详情
- **Then** 返回折扣明细字段，包含折扣类型和金额
